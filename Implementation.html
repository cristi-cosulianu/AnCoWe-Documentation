Implementation Details

For the game we used a AABB algorithm(axis aligned bounding box) so we can check collision for enviormental objects.We used a structure in which evey object in the game has his main class in the GameObject class.We had to implement our own triggers system , game controllers , gravity system and physics simulation.

At login we generate a session id on server which is passed betwwen the client and the server to identify the user. The session id is generated with a random algortihm which have a very low probability of collision.

For the database security we use md5 hashing algorithm, the password will not be avaible for the database manager, the user will be the only one who will have knowledge about the password. For the protection against SQL injection we use string escaping, any single-quote the user enters is replaced with slash followed by quotes which eliminates the user ability to end the string.

We use https security so all packages sent between server and client are encrypted.
