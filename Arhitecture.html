Abstract

We live in a world surrounded by vast amounts of information. Impressive technological advances have allowed for the volume of information we produce to grow exponentially, to the point where we now produce in two days as much information as we produced up to 2003 (that is, from the dawn of mankind to 2003)(SCH, 2010). With this in mind, we need better tools to help us comprehend vast amounts of data. One way we can do this is by visualizing the data, hence filtering it, extracting what we need and putting it into a format our visual minds are equipped to handle. Effective data visualization helps users analyze data, makes complex data more accessible, understandable and usable.

AnCoWe is a web application that involves deploying an anime-based (Anime API) RPG-style game and comic book heroes (Marvel API) to reconstitute the classic Mario gameplay scenario. Our user is able to choose more features for Mario depending on the chosen character. In addition, the user will be able to create a login account with which he can save his or her preferences for options (game keys, volume of sounds) and will be able to participate in the Hall of Fame, after completing the game his score will be entered in a list sorted with the highest scores.

Architecture

AnCoWe implemented the following MVC architecture:

Our server it's made using the open source Node.js runtime environment. We made this choice because of scalability and the relatively easy learning curve involved. Because of our architectural design choice, our app is running on a single thread and will allow hundreds of concurrent connections.

The data that is used by our app it's stored using MySQL. The connection between the database and our server is provided by mysql node module (link on npm). Basically after a user connects to our game the client makes requests for some data (login, signup, options, scores, sessions), our server will make a query to the database and then receive a response.

App flow

In mare parte orice actiune a utilizatorului, selectare de meniu, click pe login signup, taste apasate in cadrul jocului creaza requesturi specifice catre serviciu care interactioneaza cu baza de date pentru a putea construi in raspuns ce va fi procesat de catre client si va afisa rezultatul actiunilor utilizatorului. Dar cred ca diagrama este destul de sugestiva in a prezenta flow-ul aplicatiei.

Modelling the database

Baza noastra de date este formata din 4 tabele (Users, Sessions, Scores, Options). "Users" este tabela ce contine informatiile pricipale care il identifica pe utilizator si care fac legatura cu restul informatiilor despre acesta aflte in restul tabelelor. Tabela "Sessions" este utilizata pentru a identifica unic sesiunea de joc a utilizatorului si pentru a putea stoca informatiile despre jocul curent. Tabela "Scores" este utilizata pentru a tine evidenta topului cu rezultatele obtinute de jucatori, iar tabela "Options" pastreaza preferintele jucatorului legate de tastele de joc si volumul sunetelor.

REST API

Our REST API are urmatoarele path-uri /login, /signup, /scores si /options utilizand tehnologia AJAX si path-ul /data utilizand libraria "sockets.io". Au fost implementate metode de tipul GET pentru fisierele de diverse tipuri (.html, .css, .js) pentru informatiile de login, optiuni, score dar si pentru cele necesare rularii jocului. Metode de tip POST au fost folosite pentru signup si pentru optiuni.

Use Cases

This is how our use cases are looking.

Pagina principala este pagina "Menu". Ea poate fi accesata doar dupa ce utilizatorul este autentificat cu un cont existent sau dupa ce isi creaza un cont nou in pagina de "SignUP". Din pagina de "Menu" utilizatorul poate accesa meniurile de Start game, Options, Hall of Fame si Credits.

Options este meniul din care se pot modifica preferintele utilizatorului legate de tastele de joc si volumul sunetelor.

Hall of Fame este meniul in care pote fi vizualizat topul scorurilor obtinute.

Credits este meniul unde poate fi vizualizata o mica povestioara despre inceputul proiectului si de modul in care a decurs

Meniul cel mai interesant este cel de Start game care ajuta utilizatorul sa navigheze mai departe spre Choose Characters unde vor fi afisate doua liste cu caracterele Marvel sau Anime care pot fi selectate. Dupa selectarea unui caracter poate fi vizualizata o descriere a acestuia si apoi se poate naviga cate jocul propriuzis unde capacitatile jucatorului vor fi testate la maxim.
